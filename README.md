## Fast start
Для быстрого старта проекта после клонирования проекта необходимо выполнить на сервере следующие команды
```
### обновление зависимостей 
sudo apt update

### установка python и pip
sudo apt install python3.10 python3.10-venv python3-pip -y

### установка зависмотей из файла requirements.txt
pip install -r requirements.txt
```

Далее запуск самого бота
```
### выполнять из корня проекта
python main.py
```

### Docker start
Вариант описанные выше подходит для ручного старта и в большинстве случае подходит для тестирования (т.е локально), он не является масштабируемым и его нельзя отести к хорошей практики.

Для запуска в prod (продакшен) следует запускать проект в контейнере.

Образ для данного телеграмм бота находится в открытом репозитории на [docker.hub](https://hub.docker.com/repository/docker/kozlovjr02/kasedo_tg_bot/general)

Для запуска контейнера в фоново режимвы выполните следующие команды
```
### скачать образ из репозитория
docker image pull kozlovjr02/kasedo_tg_bot:latest

### запустить контейнер на основе скаченного образа, указав имя контейнеру tg_job_bot
docker run -d --name=kasedo_bot kozlovjr02/kasedo_tg_bot:latest
```

Для того, чтобы остановить контейнер для проведения регламентных работ, выполните команду

```
### остановка docker контейнера
docker stop kasedo_bot
```

### Особенности 
Данный телеграмм-бот выполняет записи в гугл-таблицу. Для этого необходимо указать данные аутетинифкации в виде json файла. Если вы хотитет, то создайте такой же файл для своих таблиц и поместите его в корень проекта (на один уровень с main.py) подробное видео как это сделать можно найти [тут](https://yandex.ru/video/preview/4256838385831369309)
